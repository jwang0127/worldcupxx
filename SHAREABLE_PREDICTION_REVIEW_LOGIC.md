# 2026 世界杯预测模型执行手册

这是一份可以直接分享给别人使用的执行文档。目标不是解释全部技术细节，而是让接手的人拿到后，能够稳定完成以下工作：

1. 更新昨日复盘
2. 生成今日或明日 4 场比赛预测
3. 输出可分享的 HTML 页面和 JSON 数据
4. 在发布前完成强制校验

## 1. 项目目标

本项目用于生成“2026 世界杯预测看板”。

每天基于以下信息生成预测内容：

1. 官方或经赛程表校验的比赛安排
2. 当日竞彩赔率
3. 已结束比赛的真实赛果
4. 外部补充数据
   - FIFA 排名
   - 伤停
   - 历史交锋
   - 小组积分榜
   - 球队资料

输出内容包括：

1. 每日预测页 `YYYYMMDD/index.html`
2. 预测副页 `predict_YYYYMMDD.html`
3. 复盘页 `review.html`
4. 评分数据 `scores_YYYYMMDD.json`
5. 比分 EV 数据 `ev_YYYYMMDD.json`
6. 小组积分数据 `standings_YYYYMMDD.json`

## 2. 根目录与关键文件

项目根目录：

`C:\Users\Administrator\Documents\世界杯预测`

首次接手前至少确认以下文件存在：

1. `2026年世界杯赛程表.xlsx`
2. `data\YYYYMMDD.json`
3. 当日赔率原始来源文件或截图
4. `team_aliases.json`

关键文件：

1. `scripts/auto_update.ps1`
   - 日常主入口
   - 负责赛程同步、外部富化、赛果回填、页面生成、复盘更新
2. `scripts/generate_daily_board.ps1`
   - 生成每日预测页
3. `scripts/sync_schedule_metadata.ps1`
   - 用赛程表校验比赛信息
4. `scripts/enrich_match_data.py`
   - 补充外部数据
5. `scripts/backfill_results.ps1`
   - 回填赛果
6. `push_to_github.ps1`
   - 推送到 GitHub
7. `team_aliases.json`
   - 中英文队名映射
8. `2026年世界杯赛程表.xlsx`
   - 最高优先级赛程基准

## 2.1 环境准备

如果要跑满外部富化数据，环境里需要配置以下变量：

1. `FOOTBALL_DATA_TOKEN`
2. `API_FOOTBALL_KEY`
3. `SERPAPI_KEY`

Windows 可用如下方式配置：

```powershell
setx FOOTBALL_DATA_TOKEN "你的 token"
setx API_FOOTBALL_KEY "你的 key"
setx SERPAPI_KEY "你的 key"
```

说明：

1. 配置后通常需要重新打开终端
2. 没有 key 时，页面仍可生成，但外部实时数据会不完整
3. 赛程表校验与本地预测主流程仍然必须执行

## 3. 最高优先级规则

### 3.1 赛程表优先于一切

以后所有预测前，必须先以 `2026年世界杯赛程表.xlsx` 为准，确认：

1. 场次编号
2. 主队
3. 客队
4. 北京时间
5. 当地时间
6. 所属小组

赔率截图只能补赔率，不能替代赛程核对。

### 3.2 玄学必须使用当地时间

玄学部分必须按比赛当地时间起卦和判断。

页面中可以同时展示：

1. 北京时间
2. 当地时间

但玄学判断只认当地时间。

### 3.3 输出前必须复查

每次输出前必须检查：

1. `data/YYYYMMDD.json` 中的主客队是否与赛程表一致
2. 北京时间是否正确
3. 当地时间是否正确
4. 小组是否正确
5. 赔率是否填入了正确场次
6. 页面中的展示是否和 JSON 一致

## 4. 这套模型现在的结构

### 4.1 主模型与玄学分离

当前模型已经改为“双轨结构”：

1. 主模型
   - 负责综合评分和主预测
2. 玄学模型
   - 单独输出
   - 不再影响综合信心分

### 4.2 主模型输出

每场比赛主模型输出：

1. 胜平负方向
2. 总进球主线
3. 半全场主推
4. 总进球区间
5. 稳胆比分 2 个
6. 冷门比分 1 个
7. 比分 EV 明细
8. 量化评分卡

### 4.3 量化评分卡维度

当前主模型主要使用：

1. 基本面
2. 阵容伤停
3. Elo
4. 赔率隐含
5. 赔率趋势
6. 历史交锋
7. 小组动机
8. 出线压力

注意：

1. 玄学不参与综合分
2. 伤停和交锋已接入外部数据管线
3. 赔率走势接口结构已预留，但还需要继续补强

### 4.4 玄学输出

当前玄学保持独立展示，包含：

1. 周易梅花 / 六爻
2. 紫微流时
3. 奇门遁甲
4. 干支五行 / 飞星
5. 诸葛神数 / 塔罗

用途：

1. 增加表达层
2. 作为娱乐型辅助说明
3. 不影响主模型综合评分

## 6. 每日标准执行顺序

### 第 1 步 确认处理日期与场次

先确认本次要处理的日期，以及 4 场比赛编号。

建议固定写成：

`今日处理：20260621，场次：033、034、035、036`

### 第 2 步 先过赛程表

打开 `2026年世界杯赛程表.xlsx`，对 4 场逐一核对：

1. 主队
2. 客队
3. 北京时间
4. 当地时间
5. 小组

### 第 3 步 检查或录入赔率

优先录入：

1. 胜平负
2. 让球胜平负
3. 总进球
4. 比分
5. 半全场

### 第 4 步 回填昨日赛果

复盘至少检查：

1. 总进球是否命中
2. 胜平负方向是否命中
3. 强队尾部是否被低估
4. 是否需要增加平局或偷球冷门防线

### 第 5 步 跑富化与生成流程

推荐使用主入口：

```powershell
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass -Force
& .\scripts\auto_update.ps1 -Date 20260621 -NoPush
```

如需先单独富化：

```powershell
C:\Users\Administrator\.cache\codex-runtimes\codex-primary-runtime\dependencies\python\python.exe .\scripts\enrich_match_data.py --date 20260621 --data-file .\data\20260621.json
```

如需只重建页面：

```powershell
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass -Force
& .\scripts\generate_daily_board.ps1 -Date 20260621
```

### 第 6 步 检查输出页

至少检查：

1. `20260621/index.html`
2. `20260621/predict_20260621.html`
3. `20260621/review.html`
4. `data/20260621.json`

重点再看四类问题：

1. 是否误把体彩场次号当赛程序号
2. 是否有主客队写反
3. 是否有北京时间正确但当地时间错误
4. 是否有赔率落错比赛

### 第 7 步 再推 GitHub

```powershell
powershell -ExecutionPolicy Bypass -File .\push_to_github.ps1
```

## 7. 本项目已经确认过的一次关键错误

### 7.1 错误内容

曾经把 `035` 写成了：

`厄瓜多尔 vs 克罗地亚`

实际应为：

`厄瓜多尔 vs 库拉索`

### 7.2 根因

不是单纯笔误，而是流程错误：

1. 没先以赛程表校对主客队
2. 把体彩场次编号误当成赛程顺序编号
3. 人工补录时没有做最终复核

### 7.3 固定修正

以后必须固定按这个顺序：

1. 先赛程校验
2. 再录赔率
3. 再生成分析
4. 最后强制复查

## 8. 对外分享时的推荐结构

如果要发给别人，建议内容结构固定如下：

### 今日四场结论

每场一行，例如：

`033 荷兰 vs 瑞典：3球，2:1 / 3:0，防 1:1`

### 今日最稳

写最有把握的一场或一个主线。

### 今日最大冷门

写最需要防守的一场平局或偷分比分。

### 昨日复盘

写清：

1. 哪几场命中
2. 哪几场偏差
3. 今日修正了什么

### 风险提示

固定保留：

`以上仅为公开信息整理后的娱乐分析，不构成任何购彩建议，请理性参考。`

## 9. 给下一位执行者的简短提醒

这套模型最重要的不是“猜比分”，而是：

1. 先把赛程核准
2. 再用赔率定主线
3. 再用复盘修正偏差
4. 最后用冷门防线防止机械跟热

只要有人要人工补录比赛，先看赛程表，再看赔率图，最后才允许出稿。

## 10. 新对话或新执行者的起步模板

如果换一个人，或者新开一个对话，建议直接按下面顺序开始：

1. 先读本文件
2. 再读 `NEXT_CHAT_MEMORY.md`
3. 确认今天要处理的日期与 4 场编号
4. 打开 `2026年世界杯赛程表.xlsx` 核对赛程
5. 检查 `data\YYYYMMDD.json` 是否已存在并与赛程一致
6. 再执行 `scripts\auto_update.ps1`
7. 输出完成后做最终复查，再推 GitHub

推荐开场指令：

`继续执行今天的世界杯预测，请先按赛程表核对 4 场比赛，再更新复盘和预测页面。`
