第一层：数据输入

所有实时接口先落入统一 Match Snapshot，保留来源、更新时间、可信度和冲突标记。

FIFA 排名动态权重、洲际强度、近年大赛修正。
比赛统计xG、xGA、射门质量、控球推进、定位球。
球员状态伤停、疲劳、首发概率、旅途与轮换。
赛事情境东道主、天气、海拔、湿度、小组动机。
赔率市场胜平负、总进球、比分、盘口异动。

第二层：特征工程

把离散赛程、实时比分、赔率和球队资料转换成模型可比较的强弱信号。

Elo 评分系统胜负实时更新 K 因子，大赛权重更高。
泊松进球模型进攻强度 × 防守强度，输出 0-7+ 分布。
对阵历史编码H2H 胜率、近 5 场趋势、风格克制。
赔率隐含概率去水后概率、冷热偏差、诱盘风险。
临场状态向量红黄牌、伤退、比分压力、剩余时间。
↓

第三层：集成模型

用多模型互相校验，强项不同：贝叶斯负责不确定性，树模型负责非线性，序列模型负责走势。

贝叶斯层级模型先验 + 实时数据，不确定性可量化。
XGBoost / RF识别赔率、体能、战术之间的非线性关系。
神经网络LSTM/Transformer 捕捉长期状态曲线。
市场校准器把模型概率和赔率概率做融合校准。
异常检测器识别赔率跳动、阵容突变、爆冷前兆。
↓

第四层：蒙特卡洛赛程模拟

对单场、当天四场、小组出线和淘汰赛路径做大量采样，输出置信区间而不是单点结论。

10 万次模拟完整赛程展开，记录比分和积分路径。
点球专项模块淘汰赛加入门将扑救率与主罚顺序。
动态位置更新实时比分改变小组排名和后续动机。
置信区间80% / 95% 区间，避免过度确定。
组合风险三串一相关性、同向风险和回撤提示。
↓

第五层：预测输出

输出给每日 HTML 看板：先给可买汇总，再给每场解释、稳胆、冷门、组合和复盘指标。

冠军概率分布32/48 队路径排名与阶段晋级率。
单场胜平负概率胜平负、让球、总进球和比分概率。
模型评估指标Brier 分数、LogLoss、命中率、回撤。
复盘反馈赛后自动结算，更新 Elo 与误差归因。
网页发布每日 13:00 生成、提交、推送 GitHub Pages。

落地到当前项目

  • 每日赔率截图或 API 数据进入 `data/YYYYMMDD.json`。
  • 生成器读取胜平负、总进球、比分与结果字段。
  • 页面展示“今天该买什么”、四场卡片、组合推荐和复盘。
  • 赛果接口确认后，`result` 字段自动回填，复盘页自动结算。

推荐实时数据闭环

主源：API-Football 备源：football-data.org 兜底：SerpApi 赛程镜像：worldcup26.ir 资料补充：TheSportsDB
  • 同场比赛至少两源一致才自动结算。
  • 只要比分源冲突，就进入“人工确认”队列。
  • 赔率和赛果分开存，避免赛果抓取失败影响预测生成。
